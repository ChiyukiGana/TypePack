# TypePack

一个 JSON 风格的 C++ 类型序列化工具，参考MessagePack，支持对象、数组及常用类型，通过二进制存储，高效紧凑。

## 存储方式

| 类型     | 存储格式                          |
|----------|-----------------------------------|
| Bool     | 8b (type)                         |
| Int      | 8b (type) + 8~64b (value)         |
| String   | 8b (type) + 8~64b (length) + data |
| Binary   | 8b (type) + 8~64b (length) + data |
| Object   | 8b (type) + 8~64b (length) + ...  |
| Array    | 8b (type) + 8~64b (length) + ...  |

### 设计特点

- 序列化时采用**最小化存储**，如 `127` 会存为 `int8`，`"abcde"` 会存为 `string8`。
- 没有大括号，方括号，逗号，引号，在对象存储的空间节省显著。
- 二进制数据直接存储，无需使用 Base64 编码。

## 空间对比

| 场景                | 格式         | 值        | 存储大小  |
|---------------------|--------------|-----------|-----------|
| 布尔值              | JSON         | true      | 4B        |
|                     | TypePack     | 1         | 1B        |
| 小整数              | JSON         | 127       | 3B        |
|                     | TypePack     | 127       | 2B        |
| 大整数              | JSON         | 2147483647       | 10B        |
|                     | TypePack     | 2147483647       | 4B        |
| 短字符串            | JSON         | "abc"     | 5B        |
|                     | TypePack     | abc       | 4B        |

## 性能测试

### 测试步骤

1. **Step-1**：1000 次对象插入，索引长度 = 4 字节，数据长度 = 20 字节
2. **Step-2**：1000 次数组插入，数据长度 = 20 字节
3. **Step-3**：10000 次序列化
4. **Step-4**：10000 次反序列化

### 测试结果

| 操作                     | 工具                  | 耗时       | 存储大小   |
|--------------------------|-----------------------|------------|------------|
| 10000 次序列化           | TypePack              | 2178 ms     | 48,913 B  |
|                          | Qt6.9.0 QJson         | 4984 ms    | 72,936 B|
| 10000 次反序列化         | TypePack              | 2417 ms    | ~44,000 B  |
|                          | Qt6.9.0 QJson         | 6443 ms    | ~44,000 B|
